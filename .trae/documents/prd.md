## 1. 产品概述

二维码依赖关系图谱分析工具，用于自动检测二维码之间的链式跳转引用关系（一个二维码的 targetUrl 指向另一个二维码的短码 `/r/{shortCode}`），通过可视化依赖网络图谱识别循环引用、孤立节点等异常，并提供影响范围分析和级联更新建议。
- 目标用户：二维码管理系统的运维人员、开发人员
- 核心价值：避免二维码更新导致的级联故障，提前发现循环跳转和死链问题

## 2. 核心功能

### 2.1 功能模块

1. **依赖图谱页面**：力导向图可视化展示所有二维码节点及引用关系边，支持缩放/拖拽/节点交互
2. **异常检测面板**：自动识别循环引用链路、孤立节点、断链引用，以不同颜色/样式标记
3. **影响范围分析**：选中节点后展示其上游/下游依赖链，计算级联影响范围
4. **级联更新建议**：针对修改/删除操作，给出受影响节点列表和建议操作步骤

### 2.2 页面详情

| 页面名称 | 模块名称 | 功能描述 |
|----------|----------|----------|
| 依赖图谱 | 图谱可视化区 | Canvas 力导向图，节点可拖拽/点击，边表示跳转关系，异常节点高亮 |
| 依赖图谱 | 侧边分析面板 | 点击节点后展示该节点详情、上下游依赖链、影响范围、级联建议 |
| 依赖图谱 | 异常概览栏 | 顶部统计卡片展示循环引用数、孤立节点数、断链引用数、总依赖关系数 |
| 依赖图谱 | 图例与筛选 | 图例说明颜色含义，支持按异常类型筛选显示 |

## 3. 核心流程

1. 用户进入依赖图谱页面，系统自动加载所有二维码数据
2. 后端分析所有 QR 码的 targetUrl，匹配 `/r/{shortCode}` 模式建立依赖边
3. 使用 Tarjan 算法检测强连通分量（循环引用），标记孤立节点
4. 前端渲染力导向图，不同类型节点使用不同颜色
5. 用户点击节点查看影响分析，或点击异常项跳转到对应节点

## 4. 用户界面设计

### 4.1 设计风格
- 主色调：延续现有深色主题，品牌蓝(#1677FF)渐变
- 循环引用节点：红色(#FF4D4F)描边 + 脉冲动画
- 孤立节点：灰色(#64748B)
- 正常依赖节点：品牌蓝渐变
- 断链引用边：橙色虚线(#FA8C16)
- 图谱背景：网格点阵(dot-pattern)

### 4.2 页面设计概览

| 页面名称 | 模块名称 | UI 元素 |
|----------|----------|---------|
| 依赖图谱 | 异常概览栏 | 4 个统计卡片（循环引用/孤立节点/断链引用/总依赖数） |
| 依赖图谱 | 图谱可视化区 | Canvas 力导向图，节点圆角矩形+图标，边带箭头 |
| 依赖图谱 | 侧边分析面板 | 滑出式面板，节点信息+依赖树+影响范围+建议操作 |
| 依赖图谱 | 图例与筛选 | 底部浮动图例，顶部下拉筛选 |

### 4.3 响应式设计
- 桌面端优先：图谱全宽展示，侧边面板右侧滑出
- 平板端：图谱缩小，面板覆盖式
- 移动端：图谱全屏，面板底部抽屉式
